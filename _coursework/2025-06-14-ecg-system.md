这个是一个心电的检测系统，基于STM32F407开发板、ADS1296采集芯片、LCD显示模块、蜂鸣器等。

## 项目概述
### 背景与目标
- 背景：针对心电信号（ECG）的实时监测需求，解决基线漂移、工频干扰、高频噪声等信号污染问题，实现精准的R波检测与心率计算。
- 目标：设计一套上下位机协同的ECG检测系统，完成信号采集、预处理、特征提取、数据分析与可视化，满足实时性、稳定性与准确性要求。

### 系统组成
- 上位机：基于Python开发，负责串口数据接收、多线程信号处理、R波检测、心率计算及图形化展示（时域/频域波形）。
- 下位机：以STM32F407开发板为核心，外接ADS1296生物电采集芯片、LCD显示屏、蜂鸣器，负责信号采集、预处理、FFT分析及数据上传。

### 输入输出与关键指标
- 输入：人体心电信号（通过ADS1296差分采样）。
- 输出：原始/滤波后ECG波形、R波峰值、实时心率、各阶段频谱图（上位机）；波形滚动显示、心跳提示音（下位机）。
- 关键指标：
  - 采样精度：ADS1296 24位ADC，可编程增益。
  - 处理延迟：上位机GUI刷新周期10ms，批处理更新间隔0.05秒。
  - 滤波参数：高通IIR（<0.5Hz基线漂移）、50Hz陷波、FIR低通（35Hz截止）、巴特沃斯带通（5-15Hz）。
  - 心率计算：基于RR间期，时间戳精度依赖DWT计时器（纳秒级）。
  - 滑动窗口：上位机700点，下位机161阶FIR滤波缓存。

## 系统架构
### 数据流与模块划分
完整数据流：
1. 采集：ADS1296差分采样心电信号，通过SPI传输至STM32。
2. 传输：下位机按自定义协议通过串口向上位机发送原始ECG数据。
3. 解析：上位机串口线程解析数据包，提取ECG值与时间戳。
4. 滤波：上下位机均执行多级滤波（IIR高通+FIR低通+陷波）。
5. R波检测：基于简化Pan-Tompkins算法识别R波位置。
6. 心率计算：通过R波时间戳计算RR间期，转换为每分钟心跳数（bpm）。
7. 显示/存储：上位机GUI展示时域/频域波形，下位机LCD滚动显示波形+蜂鸣器提示。

### 通信协议
- 帧格式：固定包头+数据段，无校验字段（简化设计）。
  - 包头：1字节，固定为0XA5（用于数据包同步）。
  - 数据段：4字节，浮点型ECG电压值（小端存储）。
- 关键字段说明：
  - 包头0XA5：用于区分有效数据包，解决传输中数据错位/丢失问题。
  - 4字节浮点数：由下位机ADS1296采样值换算得出（公式：\(V_{in }=\frac{ Data _{Raw }}{2^{23}} * \frac{V_{ref }}{ Gain }\)）。

## 上位机（PC）
### 功能列表
1. 串口接收与解析：
   - 非阻塞监听指定串口，基于包头+数据长度解析有效数据包。
   - 异常处理：支持缓冲区刷新、解包失败恢复。
2. 多线程/并发设计：
   - 三大独立线程：串口读取线程（I/O密集）、数据处理线程（计算密集）、GUI线程（显示密集）。
   - 线程同步：通过队列（raw_data_queue/processed_data_queue）传递数据，通过lock（batch_data_lock/r_peak_params_lock）保护共享变量。
   - 退出机制：通过stop_event事件+pause_condition条件变量，确保线程安全终止。
3. 滤波与检测：
   - 多级滤波：一阶IIR高通（去基线漂移）→50Hz IIR陷波（去工频干扰）→35Hz FIR低通（去高频噪声）。
   - R波检测：简化Pan-Tompkins算法+动态阈值调整。
4. GUI与图形化展示：
   - 双窗口设计：时域窗口（原始波形、滤波波形、心电图机式滚动显示）、频域窗口（4个处理阶段频谱图）。
   - 交互功能：暂停/恢复数据处理、一键终止所有进程。

### 关键算法与实现
1. 高通IIR（去基线漂移）：
   - 类型：一阶巴特沃斯高通滤波器。
   - 目的：抑制<0.5Hz的低频干扰（呼吸、电极移动导致的基线漂移）。
   - 实现：通过`scipy.signal.lfilter`调用预设计系数（b_dc/a_dc）。
2. 低通FIR（去高频噪声）：
   - 类型：161阶FIR滤波器（firls设计）。
   - 截止频率：35Hz，抑制肌电噪声、量化噪声。
   - 实现：基于滑动窗口缓存，逐点卷积计算滤波结果。
3. R波检测（简化Pan-Tompkins）：
   - 核心步骤：
     1. 5-15Hz巴特沃斯带通滤波（突出QRS波群）。
     2. 五点微分滤波（增强QRS波陡峭斜率）。
     3. 信号平方（突出高幅值QRS波）。
     4. 150ms滑动窗口积分（平滑信号，形成明显峰值）。
     5. 峰值检测：`scipy.signal.find_peaks`（基于height/distance参数）。
     6. 回溯定位：从积分信号峰值回溯至原始信号，前后100ms窗口找最大峰值作为R波。
4. 心率计算：
   - 计算逻辑：至少2个R波时，通过时间戳计算RR间期（time_r2 - time_r1），心率=60/RR间期（秒）。
   - 优化：基于时间戳而非采样点序号，解决采样不均匀导致的误差。
   - 动态调整：根据心率实时修改R波最小间隔（心率>150时缩小间隔，<50时增大间隔，范围0.2-1.0秒）。

## 下位机（STM32F407 + ADS1296）
### 硬件组成
| 硬件模块       | 核心参数与功能                                                                 |
|----------------|------------------------------------------------------------------------------|
| MCU（STM32F407）| 32位Cortex-M4内核（含FPU），主频168MHz，210 DMIPS；1MB Flash+192KB SRAM；支持SPI/DMA/外部中断。 |
| 采集芯片（ADS1296）| 6通道24位ADC，集成ECG前端放大器、右腿驱动（RLD）、导联脱落检测；SPI通信，DRDY引脚触发数据读取。 |
| 显示模块（LCD） | 实时滚动显示ECG波形，60Hz帧率，局部刷新技术避免闪烁；X轴对应时间，Y轴对应电压幅度。         |
| 提示模块（蜂鸣器）| 检测到R波时触发1-2kHz PWM信号，发声50-100ms；200ms不应期防止连续发声。                 |
| 通信模块（串口）| UART/USB转串口，用于向上位机发送ECG数据，波特率【待补充】。                             |

### 信号处理流程
1. 采集配置：
   - ADS1296配置：通过SPI设置采样率、增益、偏置，启动ΔΣ转换。
   - 数据读取：DRDY引脚下降沿触发外部中断，STM32在中断服务程序中读取24位ADC值（MSB优先）。
   - 导联检测：通过ADS1296状态字判断电极接触状态（左右导联是否正常）。
2. 滤波、检测、心率测量：
   - 滤波流程：一阶IIR高通（去基线漂移）→161阶FIR低通（40Hz截止）。
   - R波检测：简化Pan-Tompkins算法（差分→绝对值/平方→150ms滑动窗口积分→自适应阈值判断）。
   - 心率测量：通过Cortex-M4内核DWT计时器（CYCCNT寄存器，CPU时钟计数）记录R波时间戳，精度达纳秒级；RR间期换算心率。
   - 不应期设计：200ms内忽略重复峰值，避免QRS波多次检测。
3. FFT/频谱分析：
   - 实现：基于CMSIS-DSP库的`arm_rfft_fast_f32`函数，实时转换时域信号至频域。
   - 用途：分析信号频率成分（1-2Hz心跳节律、50Hz工频噪声），评估滤波效果。

## 实验与结果
### 测试设置
- 数据来源：人体实际心电信号（通过ADS1296差分导联采集）。
- 工况：静态采集环境，无剧烈运动，避免强电磁干扰。
- 对比方式：通过上位机显示的滤波前后波形/频谱对比，验证噪声抑制效果；通过R波检测准确率、心率稳定性评估系统性能。

### 结果展示
1. 上位机结果：
   - 时域窗口：原始信号基线漂移明显，滤波后QRS波群清晰；R波峰值标注准确，心率显示实时更新（更新周期4秒）。
   - 频域窗口：原始信号含50Hz工频尖峰，陷波后该尖峰显著抑制；低通滤波后高频噪声能量大幅降低。
   - 图1：上位机PC端心电数据分析结果（含原始信号、滤波后信号、R波峰值及各阶段频谱图，波形清晰准确）。
2. 下位机结果：
   - LCD显示：ECG波形滚动流畅，无闪烁；R波位置与上位机检测结果一致。
   - 蜂鸣器提示：发声频率与心率同步，无漏报/误报。
   - 图2：下位机滤波前/后频谱对比图（滤波后50Hz噪声峰值消失，低频心跳成分保留）。
3. 核心结论：系统运行稳定，信号处理效果符合预期，R波检测准确率高，心率计算实时性满足需求。

### 误差与问题分析
- 心率误差：未明确量化数据，推测误差来源为采样噪声、电极接触不良导致的信号失真【待补充具体误差值】。
- 潜在问题：
  - 上位机：多线程高负载时可能出现GUI卡顿（未实测验证）。
  - 下位机：导联脱落时无明确报警提示，仅内部状态标记。
  - 通信：无数据校验机制，极端情况下可能出现数据包丢失。

## 亮点与工程经验
### 实时性/稳定性设计要点
1. 多线程架构：将I/O密集（串口读取）、计算密集（信号处理）、显示密集（GUI）分离，解决“数据I/O阻塞UI”问题，使GUI更新速度提升3倍。
2. 数据包同步：基于包头0XA5+固定长度解析，配合缓冲区刷新机制，提升通信鲁棒性。
3. 动态参数调整：R波检测最小间隔随心率自适应变化，平衡漏检与误报。
4. 高精度计时：下位机采用DWT计时器，解决传统定时器精度不足问题，提升心率计算准确性。
5. 滤波优化：上下位机协同滤波，下位机预处理减少上位机计算压力，上位机精细化滤波提升显示效果。

### 跨平台协作与接口定义
1. 协议规范化：统一上下位机数据帧格式（包头+4字节浮点数），使数据传输与解析无歧义。
2. 接口解耦：上下位机通过串口通信，硬件/软件修改互不影响（如上位机更换PC、下位机更换MCU不影响协作）。
3. 数据一致性：上下位机采用相同的ECG电压换算公式，确保数据语义统一。

## 后续改进
1. 算法优化：
   - 增强Pan-Tompkins算法：补充完整自适应阈值与回溯机制，提升复杂噪声环境下R波检测准确率。
   - 滤波参数自适应：根据信号噪声强度动态调整滤波器截止频率/增益。
2. 协议健壮性：
   - 增加校验字段（如CRC8/CRC16），解决传输误码问题。
   - 增加重传机制：上位机未收到有效数据包时，向下位机发送重传请求。
3. UI/交互体验：
   - 上位机：增加心率异常报警（如心率>150或<50时弹窗提示）、历史数据存储与回放功能。
   - 下位机：LCD增加心率数值显示、导联脱落报警图标。
4. 硬件扩展：
   - 增加电池供电模块，实现系统便携化。
   - 扩展多导联采集（ADS1296支持6通道），提升信号采集灵活性。
5. 异常处理：
   - 上位机增加串口重连机制，避免串口断开后程序崩溃。
   - 下位机增加低功耗模式，延长续航（针对便携场景）。
